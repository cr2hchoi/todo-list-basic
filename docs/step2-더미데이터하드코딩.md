# 🔍 Step 2: 입력 UI 및 리스트 더미데이터 구조 분석

Bootstrap 컴포넌트를 활용하여 할 일 입력창과 목록의 정적 레이아웃을 구성함.

### 💡 하드코딩(Hard-coding)을 선행하는 이유

실제 동적 데이터를 바인딩하기 전, 정적 데이터를 직접 입력하여 기초를 다지는 과정임.

1. **UI/UX 검증**: 디자인 요소(Bootstrap 클래스 등)가 의도대로 렌더링되는지 즉각 확인 가능함.
2. **데이터 모델 설계**: 화면에 필요한 정보(내용, 상태, ID 등)를 파악하여 향후 자바스크립트 객체 구조를 설계하는 가이드라인이 됨.
3. **디버깅 용이성**: 정적 레이아웃이 완성된 상태에서 로직을 추가해야 화면 렌더링 오류와 로직 오류를 분리하여 파악할 수 있음.

### 1. 입력부 구조 (`input-group`)

- **`.row`, `.mb-3`**: 행(Row)을 생성하고 하단 여백(Margin Bottom)을 주어 목록과의 간격을 확보함.
- **`.input-group`**: 입력창(`input`)과 버튼(`span`)을 하나로 결합하여 일체감 있는 UI 구현함.
- **`placeholder`**: 입력 전 안내 문구("할 일을 입력해주세요!!")를 표시하여 사용자 편의성 높임.
- **`.btn-primary`**: 파란색 강조 버튼을 적용하여 '추가' 액션을 시각적으로 두드러지게 함.

### 2. 출력부 구조 (`list-group`)

- **`.list-group`**: 목록 형태의 데이터 출력을 위한 컨테이너 활용함.
- **`.list-group-item`**: 개별 할 일 항목을 나타내는 단위임.
- **상태별 시각화**:
  - **완료 상태**: `.list-group-item-success` 클래스로 배경색(초록색) 변경 및 `.todo-done`으로 취소선 적용함.
  - **진행 상태**: 기본 테마 유지함.
- **`.float-end`**: 삭제 버튼(Badge)을 우측 정렬하여 텍스트와 겹치지 않게 배치함.
- **`.badge`**: 삭제 기능을 텍스트가 아닌 알약 형태의 UI 컴포넌트로 시각화함.
- **`.pointer`**: 모든 클릭 가능 요소에 커서 효과를 부여하여 상호작용 가능함을 명시함.
