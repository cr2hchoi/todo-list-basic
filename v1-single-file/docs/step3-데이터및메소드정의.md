# 🔍 Step 3: 데이터 구조 및 비즈니스 로직 정의

Vue.js 인스턴스를 생성하고, 앱 구동에 필요한 데이터 객체와 메서드를 설계함.

### 1. 데이터 모델 (Data Model)

- **`ts (Timestamp)`**: 전역 변수로 현재 시간을 측정하여 고유 키(ID) 생성 시 기준으로 활용함.
- **`todo`**: 사용자의 실시간 입력값을 담는 문자열 변수임.
- **`todolist`**: 할 일 객체들을 담은 배열.
  - `id`: 고유 식별자.
  - `todo`: 할 일 텍스트.
  - `completed`: 완료 여부(Boolean).

### 2. 핵심 메서드 (Methods) 분석

- **`addTodo()`**:
  - 유효성 검사(2자 이상) 후 `push`를 통해 배열 끝에 새 항목 추가함.
  - 추가 후 입력창을 비우기 위해 `this.todo = ''` 처리함.
- **`deleteTodo(id)`**:
  - `findIndex`를 통해 해당 ID의 인덱스 번호를 탐색함.
  - `splice(index, 1)`를 사용하여 원본 배열에서 해당 항목을 정확히 제거함.
- **`toggleCompleted(id)`**:
  - 논리 부정 연산자(`!`)를 활용하여 `completed` 상태를 반전(True ↔ False)시킴.

### 💡 Vue.js 특이사항 기록

- **Reactivity (반응형)**: Vue의 Proxy 시스템 덕분에 배열이나 객체를 수정하면 화면이 자동으로 갱신됨. 별도의 DOM 조작이 필요 없음.
- **`this` 바인딩**: 메서드 내부에서 `data`에 정의된 변수에 접근할 때 `this` 키워드 사용이 필수적임.

### ⚠️ 현재 상태 점검: 데이터와 화면의 분리

- **현상**: 스크립트에서 `todolist` 데이터를 정의했으나, 라이브 서버 실행 시 화면은 여전히 하드코딩된 정적 텍스트만 출력됨.
- **원인**: HTML 요소와 Vue 인스턴스의 데이터 간에 **'디렉티브(Directive)'**를 통한 연결 작업이 이루어지지 않음.
- **해결 방안**:
  - `v-model`을 사용하여 입력창과 데이터 동기화함.
  - `v-for`를 사용하여 배열 데이터를 기반으로 리스트 동적 생성함.
  - `v-bind` 및 `@click`을 사용하여 속성과 이벤트를 연결함.
